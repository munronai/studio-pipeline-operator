# reference/ — The Operator's Working Library

Pull these **as needed**, per the routing table in `CLAUDE.md`. **Do not preload.** Each file does one job. Open the one the station needs, run it, close it.

| File | What it is | Pull when |
|---|---|---|
| `the-pipeline.md` | The 7 stages in funnel order, OPEN_STAGES, the funnel math (10–15% end-to-end) | You need to locate a lead's station or quote conversion math |
| `gate-checklist.md` | The hard auto-disqualify gate — excluded industries + services-we-don't-offer, verbatim | A raw lead arrives, before any scoring |
| `scoring-rubric.md` | The 5 dimensions, weights, 0–10 anchors, composite formula, grade thresholds, grade→routing, intake 5-signal bands | You need to score a gated lead or justify a grade |
| `walk-away-flags.md` | 7 hard flags, soft walk-aways, lean-in signals, the decline script | A flag might be tripped, or a lean-in signal appears |
| `qualifier-protocol.md` | The 5 qualifier questions, budget/timeline branches, proceed/decline/refer rules | Qualifier answers come back |
| `diagnostic-protocol.md` | The 60-min diagnostic structure + the key questions | Diagnostic findings come back, or you draft the diagnostic-confirm |
| `service-routing.md` | Patterns A–H, the 4 tiers, the pricing decision tree, milestone splits | You route a diagnostic to a prescription + price |
| `outreach-templates.md` | Cold-call angles 1/2/3, the 60-sec qualifier, the booking close, cold email, follow-ups, warm-network, decline + referral — paste-ready | You draft any artifact |
| `pipeline-fields.md` | The exact STAGES / OUTCOMES / DECLINE_REASONS / CHANNELS taxonomy | You write a PIPELINE UPDATE line and need valid values |

The spine itself (`CLAUDE.md`, `identity.md`, `rules.md`) is always loaded. These references are the detail the spine points at.
