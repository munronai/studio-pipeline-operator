The problem

The founder is drowning in triage.

A solo studio founder sources 40–60 leads a week — cold calls, cold email, walk-ins, referrals, inbound forms. The selling isn't what kills them. The triage is: deciding what each lead is worth, what to send, and what happens next, across dozens of leads stalled at different stages at once.

WARN47 leads open across 6 stages — no disposition on 31 of them
STALLApex HVAC · contacted — no retry logged for 3 days
STALLCrestline Dental · proposal — silent 5 days, no follow-up drafted
QUEUE9 scraped leads ungraded — the gate hasn't run
LOSS~half the workday spent deciding, not selling
The Disposition Engine

Six stations. One engine.

Every input runs the same loop: locate the station → apply that station's rule → emit a disposition + a drafted artifact + the exact pipeline write. Before any outreach, two non-negotiable gates fire — the hard-disqualify Gate, then the 5-dimension Score.

1
Gate & Grade
→ new
Run the gate. If it clears, score it. Route by grade. Draft the first touch.
2
Outreach Outcome
→ contacted / replied
Read the touch result. Voicemail, retry, book the qualifier, or kill.
3
Qualify
→ qualifier
5 answers in. Proceed, decline on a hard flag, or refer below floor.
4
Diagnose & Prescribe
→ diagnostic
Route to a pattern A–H, set the tier and price, draft the proposal.
5
Close / Recover
→ proposal
Accept → kickoff. Objection → counter. Silence → 2 follow-ups, then ghosted.
6
Loss / Recovery
→ lost
Capture the reason. Nurture re-entry in 60–90d, or archive.

The output shape — every response, every station

WhereThe lead's current station + what just happened. One line.
DispositionThe decision, stated first, with the rule that fired and the grade.
ArtifactThe drafted thing — call angle, email, decline — ready to send. Actual words, not "you could say…".
Next action + dueThe single next thing and when it's due.
Pipeline updateThe exact field writes from the real taxonomy: stage · grade · channel · outcome.
FlagsOnly when present. What's escalated to the human, why, and the one open question.

Grounded, not vibes

Every decision is real logic.

The disposition you just watched run is not improvised. It's the studio's actual decision system — a weighted rubric, hard gate filters, and walk-away flags — encoded explicitly.

5-signal rubricΣ = 100
  • Need — weak / missing site30%
  • ICP Fit — local service, right size25%
  • Reach — phone, named owner20%
  • Pay — established, busy15%
  • Intent — growing now10%

composite = Σ(dim/10 × weight) → A+≥90 · A 80 · B 65 · C 50 · D<50
