MECHANISM // THE DISPOSITION ENGINE

Six stations. One engine.

Every input runs the same loop: locate the station → apply that station's rule → emit a disposition + a drafted artifact + the exact pipeline write. Before any outreach, two non-negotiable gates fire — the hard-disqualify Gate, then the 5-dimension Score.

STATION 01
Gate & Grade
→ new
Run the gate. If it clears, score it. Route by grade. Draft the first touch.
STATION 02
Outreach Outcome
→ contacted / replied
Read the touch result. Voicemail, retry, book the qualifier, or kill.
STATION 03
Qualify
→ qualifier
5 answers in. Proceed, decline on a hard flag, or refer below floor.
STATION 04
Diagnose & Prescribe
→ diagnostic
Route to a pattern A–H, set the tier and price, draft the proposal.
STATION 05
Close / Recover
→ proposal
Accept → kickoff. Objection → counter. Silence → 2 follow-ups, then ghosted.
STATION 06
Loss / Recovery
→ lost
Capture the reason. Nurture re-entry in 60–90d, or archive.

THE OUTPUT SHAPE — every response, every station

WHEREThe lead's current station + what just happened. One line.
DISPOSITIONThe decision, stated first, with the rule that fired and the grade.
ARTIFACTThe drafted thing — call angle, email, decline — ready to send. Actual words, not "you could say…".
NEXT ACTION + DUEThe single next thing and when it's due.
PIPELINE UPDATEThe exact field writes from the real taxonomy: stage · grade · channel · outcome.
FLAGSOnly when present. What's escalated to the human, why, and the one open question.

A few options: you could reach out about a redesign, pitch AI tools, or add them to a nurture list. What would you like to do? I can draft whichever you prefer."
